# Security Policy

## Reporting A Vulnerability

Do not report security vulnerabilities in public GitHub issues. Public issues are disabled for this repository.

Use GitHub private vulnerability reporting for this repository when it is available from the repository **Security** tab. If that path is not available, use the private Whooptido app support/reporting flow and include `security` in the report title.

Please include a concise description of the issue, the affected platform, the companion version or release tag if known, and the minimum reproduction details needed to understand the risk.

Do not include:

- Whooptido account details, entitlement state, or customer identifiers
- private audio, captions, transcripts, screenshots, recordings, or media URLs
- extension diagnostic logs, native messaging payloads, tokens, secrets, or service URLs
- exploit details that would put current users at immediate risk before we can triage privately

## Public Issues

Customer support and app issues belong in the private Whooptido app/reporting flow.

## Supported Scope

This repository covers the public ASR companion installer/runtime surface. Whooptido extension behavior, account access, paid entitlement state, customer media, caption output, and beta support reports are handled through the private Whooptido product support flow.
